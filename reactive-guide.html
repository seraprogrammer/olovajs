Signals ​

Signals are like magic boxes that hold values and tell everyone when they change.

Basic Signal ​

jsx
// Create a signal to store a number
const [count, setCount] = createSignal(0);

// Read the value
console.log(count()); // Shows: 0

// Change the value
setCount(1);

Effects ​

Effects watch for changes and do something when they happen.

Simple Effect ​

jsx
const [name, setName] = createSignal("Bob");

// This runs whenever name changes
createEffect(() => {
  console.log("Hello,", name());
});

setName("Alice"); // Console shows: "Hello, Alice"

Cleanup Effect ​

jsx
createEffect(() => {
  const timer = setInterval(() => {
    console.log("Tick!");
  }, 1000);

  // Cleanup when effect reruns or stops
  return () => clearInterval(timer);
});

Memos ​

Memos remember calculations and only update when needed.

Basic Memo ​

jsx
const [price, setPrice] = createSignal(10);
const [quantity, setQuantity] = createSignal(2);

// Calculates total only when price or quantity change
const total = createMemo(() => price() * quantity());

console.log(total()); // Shows: 20

Resources ​

Resources help you fetch and manage data from outside your app.

Simple Resource ​

jsx
// Function to get data
const fetchUser = async (id) => {
  const response = await fetch(`https://api.example.com/users/${id}`);
  return response.json();
};

// Create a resource
const [user] = createResource(1, fetchUser);

// Use the resource
console.log(user.loading); // Is it still loading?
console.log(user()); // Get the data
console.log(user.error); // Was there an error?

Context ​

Context lets you share information with many parts of your app.

Creating Context ​

jsx
// Create a context for theme
const ThemeContext = createContext("light");

// Provider component
function ThemeProvider(props) {
  return (
    <ThemeContext.Provider value="dark">{props.children}</ThemeContext.Provider>
  );
}

Complete Example ​

Here's everything working together:

jsx
// Create our contexts
const ThemeContext = createContext("light");
const UserContext = createContext(null);

function GameApp() {
  // Create our signals
  const [score, setScore] = createSignal(0);
  const [lives, setLives] = createSignal(3);

  // Create a memo for game status
  const gameStatus = createMemo(() => {
    if (lives() <= 0) return "Game Over";
    if (score() >= 100) return "You Win!";
    return "Playing";
  });

  // Create an effect for game events
  createEffect(() => {
    const status = gameStatus();
    if (status !== "Playing") {
      console.log(`Game Ended: ${status}`);
    }
  });

  // Create a resource for high scores
  const [highScores] = createResource(fetchHighScores);

  return (
    <ThemeContext.Provider value="dark">
      <UserContext.Provider value={{ name: "Player 1" }}>
        <div>
          <h1>Game Status: {gameStatus()}</h1>
          <p>Score: {score()}</p>
          <p>Lives: {lives()}</p>

          <button onClick={() => setScore((s) => s + 10)}>Score Points</button>

          <button onClick={() => setLives((l) => l - 1)}>Lose Life</button>

          {/* Show high scores when loaded */}
          <div>
            High Scores:
            {highScores.loading
              ? "Loading..."
              : highScores()?.map((score) => <div>{score}</div>)}
          </div>
        </div>
      </UserContext.Provider>
    </ThemeContext.Provider>
  );
}

Tips to Remember ​

  1. Signals (createSignal):

    • Are for storing changing values
    • Use () to read
    • Use setter to change

  2. Effects (createEffect):

    • Run when dependencies change
    • Good for side effects
    • Can have cleanup

  3. Memos (createMemo):

    • Remember calculations
    • Update only when needed
    • More efficient than calculating in render

  4. Resources (createResource):

    • Handle async data
    • Manage loading states
    • Handle errors

  5. Context (createContext):

    • Share data with components
    • Avoid prop drilling
    • Use providers to set values
