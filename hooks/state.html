Olova $state Hook Documentation 🎣 ​

Introduction ​

The $state hook in Olova is a lightweight implementation of React's useState hook, providing state management capabilities for functional components.

Basic Usage ​

javascript
import { $state } from "olova";

function Counter() {
  const [count, setCount] = $state(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

API Reference ​

Syntax ​

javascript
const [state, setState] = $state(initialValue);
1

Parameters ​

  • initialValue: The initial state value. Can be any valid JavaScript value (number, string, boolean, object, array, etc.).

Returns ​

Returns an array with exactly two elements:

  1. Current state value
  2. State updater function

State Updates ​

Direct Updates ​

javascript
const [count, setCount] = $state(0);
setCount(1); // Directly set new value
1
2

Functional Updates ​

javascript
const [count, setCount] = $state(0);
setCount((prevCount) => prevCount + 1); // Update based on previous state

Object State ​

javascript
function UserProfile() {
  const [user, setUser] = $state({
    name: "",
    age: 0,
  });

  return (
    <div>
      <input
        value={user.name}
        onChange={(e) => setUser({ ...user, name: e.target.value })}
      />
      <input
        type="number"
        value={user.age}
        onChange={(e) => setUser({ ...user, age: Number(e.target.value) })}
      />
    </div>
  );
}

Array State ​

javascript
function TodoList() {
  const [todos, setTodos] = $state([]);

  const addTodo = (text) => {
    setTodos([...todos, { id: Date.now(), text }]);
  };

  return (
    <div>
      <button onClick={() => addTodo("New Todo")}>Add Todo</button>
      <ul>
        {todos.map((todo) => (
          <li key={todo.id}>{todo.text}</li>
        ))}
      </ul>
    </div>
  );
}

Implementation Details ​

The $state hook in olova:

  • Manages state updates through a batching system
  • Triggers re-renders when state changes
  • Maintains state between renders
  • Handles both synchronous and asynchronous updates

Common Pitfalls ​

  1. Mutating State Directly

    javascript
    // Wrong
const [user, setUser] = $state({ name: "John" });
user.name = "Jane"; // Don't mutate state directly

// Correct
setUser({ ...user, name: "Jane" });

Performance Considerations ​

  • Use functional updates when new state depends on previous state
  • Consider using $memo or $callback hooks for expensive calculations
  • Avoid creating new objects/arrays in every render

Error Handling ​

The hook includes error handling for common issues:

  • Invalid initial values
  • Incorrect usage outside components
  • State updates on unmounted components

Related Hooks ​

  • $effect: For side effects
  • $memo: For memoized values
  • $callback: For memoized callbacks
  • $reducer: For complex state logic

Conclusion ​

The $state hook in Olova provides a simple yet powerful way to manage component state. It follows React's useState pattern while maintaining a lightweight footprint and predictable behavior.
